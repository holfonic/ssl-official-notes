# SSL Official Notes

## 项目简介

本项目是一个用于归档与发布多个独立 HTML 页面的静态仓库。所有页面均以纯前端形式呈现，不依赖后端服务，适合作为信息展示、资料存档或简单文档的托管方案。

仓库本身不针对任何特定域名或网站，仅作为页面文件的集合与版本管理之用。

## 目录说明

```
ssl-official-notes/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 页面使用的静态资源（图片、样式表、脚本等）
├── index.html      # 仓库首页（可选，用于导航）
└── README.md       # 本文件
```

- `pages/`：每个子目录或文件对应一个独立的 HTML 页面，命名清晰以便识别。
- `assets/`：按需组织，资源文件与页面逻辑分离，便于维护。
- `index.html`：若启用 GitHub Pages，可在此处提供页面索引或跳转链接。

## 页面归档说明

- 所有页面均为静态 HTML，可直接通过浏览器打开。
- 页面内容以信息展示或资料存档为目的，不包含动态交互或后端逻辑。
- 归档遵循“一页一主题”原则，页面间无强关联，各自独立。
- 页面可能使用外部 CDN 资源（如字体、图标库），但不会收集用户数据。

## 维护说明

- 欢迎提交 Issue 报告页面显示问题或内容错误。
- 若需新增页面，请将 HTML 文件放入 `pages/` 目录，并在 `index.html` 中添加导航链接（如有）。
- 不建议修改已有页面的核心内容，除非发现事实性错误。
- 本仓库不接受涉及商业推广、诱导点击或违反平台政策的页面提交。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `pages/` 下的 HTML 文件，或部署到任意静态托管服务。
3. 如需启用 GitHub Pages，可在仓库设置中指定 `main` 分支的根目录为发布源。
